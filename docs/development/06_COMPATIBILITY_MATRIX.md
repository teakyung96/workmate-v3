# 버전 호환성 매트릭스 (05_COMPATIBILITY_MATRIX.md)

> **문서 목적**: 1단계 환경 스택 인터뷰의 **캐스케이드 필터**(JDK를 먼저 묻고, 호환되는 프레임워크 버전만 선택지로 제시)와 뼈대 생성 시 **자동 파생 값**의 단일 기준표.
> **AI 지시사항**: 인터뷰에서 버전을 물을 때 반드시 이 표를 먼저 읽고, **표에서 허용되는 조합만 선택지로 제시**한다. 사용자가 표 밖의 값을 직접 입력(Other)하면 §6 검증 규칙대로 처리한다. 표의 값이 낡았을 가능성(신규 릴리스)이 있으면 그 사실을 밝히고 사용자에게 확인한다.

## 1. JDK → Spring Boot (첫 질문: JDK / 두 번째 질문: 아래 허용 목록만 제시)

| JDK | 선택지로 제시할 Boot 버전 | 선택지 설명에 병기할 문구 |
|---|---|---|
| 8 | **2.7.x** (2.x 계열만) | "Boot 3.x는 JDK 17+ 필수라 선택 불가. 2.7은 OSS 지원 종료(EOL) — 보안 패치 없음, 진행 시 주의" |
| 11 | **2.7.x** (2.x 계열만) | 위와 동일 |
| 17 | **3.5.x (권장)** / 3.0~3.4 / 2.7.x(레거시 통일 목적일 때만) | "권장: 3.5.x. 2.7은 EOL" |
| 21 | **3.5.x (권장)** / 3.2~3.4 | "Boot 3.2부터 JDK 21 공식 지원 — 3.0/3.1은 제시하지 않는다" |

## 2. Boot 메이저 → 자동 파생 값 (사용자에게 묻지 않는다 — 01 문서에 파생 변수로 기록)

| 파생 항목 | Boot 2.7.x | Boot 3.x |
|---|---|---|
| `JAVAX_JAKARTA` (import 네임스페이스) | `javax.*` (persistence/servlet/validation) | `jakarta.*` |
| `MYBATIS_STARTER_VERSION` | **2.3.x** (예: 2.3.2) | **3.0.x** (예: 3.0.3) |
| Security 설정 문법 | `SecurityFilterChain` 빈 방식 가능(Security 5.7) — 단 **경로 매처는 `antMatchers`** (`requestMatchers`는 5.8+라 **컴파일 실패**, 실측 확인). 레거시 코드베이스는 `WebSecurityConfigurerAdapter` 관례가 흔하므로 기존 코드에 추가할 땐 프로젝트 관례 확인 | `SecurityFilterChain` + `requestMatchers` (adapter는 제거됨) |
| 04 §2.4 `class file major version 65` caveat | **해당 없음 — 이 경고·wrapper 강제를 적용하지 않는다** | Boot 3.3+면 적용 |
| `spring-boot-starter-*` 좌표 | 동일 (버전만 Boot 버전 따름) | 동일 |

## 6. 검증 규칙 (게이트 동작 — 캐스케이드를 벗어난 입력 처리)

1. **불가능 조합** (예: JDK 8 + Boot 3.5): 세팅을 진행하지 않는다. 왜 불가능한지 한 줄 설명 + **양방향 대안**을 제시하고 사용자가 고르게 한다 — "JDK 8을 유지하려면 Boot 2.7 / Boot 3.5를 쓰려면 JDK 17+".
2. **가능하지만 비권장** (EOL 계열, 지원 종료 임박): 경고를 명시하고 진행 여부를 확인받은 뒤 진행한다.
3. **표에 없는 값** (신규 릴리스, 특이 버전): 임의로 판단하지 않는다. "이 표 기준으로는 확인 불가"라고 밝히고 사용자에게 호환성 근거(클라이언트 공지, 공식 문서)를 확인받는다.
4. 게이트를 통과한 최종 조합과 파생 값(§2)은 `01_PROJECT_CONFIG.md`에 기록해 이후 단계·인수인계자가 재확인할 수 있게 한다.
