# 05. 공통 요구사항 명세 (Common Requirements)

- **작성일**: 2026-07-14
- **상태**: 사용자 검토 대기
- **상위 문서**: [workmate-design.md](workmate-design.md)
- **관련 문서**: [01 기능 명세서](01_FEATURE_SPEC.md) · [04 API·DB 설계서](04_API_DB_SPEC.md)

특정 기능이 아니라 **전 기능에 걸쳐 적용되는(횡단) 요구사항**을 정의한다.
F1~F7 기능별 요구사항은 01 문서에, 코딩 방식·표준(어떻게 짤 것인가)은 `docs/setup/` 개발 가이드에 있다 — 이 문서는 그 사이의 "전 기능이 공통으로 지켜야 할 무엇"을 담는다.
각 항목은 별도 마일스톤 없이 **해당 기능이 구현되는 마일스톤에서 함께** 구현·검증한다.

## F8. 로깅 요구사항

### F8.1 원칙

- **코딩 표준**은 `docs/setup/02_DEVELOPMENT_GUIDE_BACKEND.md` 를 따른다: `@Slf4j` + `{}` 치환자, `System.out.println` 금지, 예외 시 `log.error(msg, e)`로 예외 객체 포함.
- 이 절은 표준이 아니라 **"어떤 사건을 반드시 남길 것인가"**(이벤트 로그)를 정의한다.
- **개인정보 로그 금지**: 비밀번호(평문/해시 모두)·이메일·전화번호를 로그에 남기지 않는다. 사용자 식별은 `user_seq`로만.
- 레벨 기준: 정상 흐름 추적 `INFO` / 재시도·폴백 등 비정상이지만 처리된 상황 `WARN` / 처리 불가 예외 `ERROR`.
- 관리자 감사 로그는 파일 로그와 별개로 **DB(`admin_audit_log`) 저장 확정** — 04 문서 §4.6.

### F8.2 필수 이벤트 로그

| 영역      | 남길 사건                                                  | 레벨                                 | 포함 항목                                   |
| --------- | ---------------------------------------------------------- | ------------------------------------ | ------------------------------------------- |

| F1 인증   | 중복 로그인으로 기존 세션 만료                             | INFO                                 | user_seq                                    |

| F2 채팅   | AI 호출 실패 (원인 구분: 타임아웃/쿼터/기타) + 재시도 여부 | WARN(재시도 성공) / ERROR(최종 실패) | 원인 구분 코드, 모델명                      |

| F5 Tool   | Tool 호출 (도구명·파라미터) 및 실행 오류                   | INFO / ERROR                         | user_seq, 도구명, 파라미터 요약             |

| 공통      | `GlobalExceptionHandler` 도달 예외                         | ERROR                                | 요청 URI, 예외 객체 포함                    |

### F8.3 완료 조건

- [ ] AI 호출 1건에 대해 시작→완료(모델명·소요시간) 로그가 짝으로 남음

## F9. 서버 측 입력 검증 공통 원칙

### F9.1 대원칙

**클라이언트 검증은 UX 보조일 뿐, 방어선은 항상 WAS다.** 모든 외부 입력은 WAS 진입 시점에 정규화·검증을 거친 값만 비즈니스 로직·저장에 사용한다. (배경: 기존 프로젝트에서 클라이언트가 대문자 입력을 막았는데도 우회 입력이 그대로 저장된 버그 경험 — F1-01a 이메일 정규화와 같은 부류의 문제를 전 입력으로 일반화)

### F9.2 검증 항목

| ID    | 대상                    | 규칙                                                                                                                                                         | 위험도 |
| ----- | ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------ |
| F9-01 | 이메일 (F1)             | `trim` + 소문자 정규화 후 중복검사·암호화·저장·대조 (= F1-01a)                                                                                               | 🔴     |
| F9-02 | 사업자번호 (F3)         | 숫자 외 문자(하이픈·공백) 제거 후 10자리 검증 → 체크섬 → 저장. 수정 폼(F3-08a) 입력도 동일 경로                                                              | 🔴     |
| F9-03 | 결제일·금액 (F3)        | 결제일: 숫자만 추출 후 `YYYYMMDD` 실존 날짜 검증(예: 13월 거부). 금액: 콤마·원 제거 후 양의 정수 검증                                                        | 🔴     |
| F9-04 | modelCode (F2·F7)       | 공통코드 `AI_MODEL`의 `use_yn=true` 코드만 허용 — **화이트리스트 검증**. 목록 외 값은 400 (임의 모델 주입 차단)                                              | 🔴     |
| F9-05 | 마크다운 렌더링 (F2·F4) | AI 응답·가이드 본문 렌더링 시 새니타이즈 — `<script>`·이벤트 핸들러 속성 등 위험 태그 제거. AI 응답도 프롬프트 인젝션 경유 악성 HTML 가능성이 있어 동일 적용 | 🔴     |
| F9-06 | CSV 내보내기 (F3-12)    | 셀 값이 `=` `+` `-` `@` 로 시작하면 선행 `'` 이스케이프 — 엑셀 수식 인젝션 차단. `card_name`(AI 추출 텍스트)이 주 대상                                       | 🟡     |
| F9-07 | 업로드 파일 (F3-01)     | 확장자 검사에 더해 **매직 바이트**(파일 형식 서명) 검증 — 확장자만 바꾼 위장 파일 차단. 저장 파일명은 UUID 생성 유지                                         | 🟡     |
| F9-08 | 소유권 검증 일반화      | **모든 `{id}` 경로 파라미터는 소유권 검증** — 영수증(F3-14)뿐 아니라 채팅방(roomSeq)·가이드(guideSeq)도 타인 자원 접근 시 403. 서비스 계층 공통 규칙         | 🔴     |
| F9-09 | 전화번호 (F1)           | 숫자 외 문자 제거 후 암호화·저장                                                                                                                             | 🟢     |
| F9-10 | 문자열 길이 상한        | 채팅 메시지·가이드 제목/본문 등 서버 측 최대 길이 검증 (한도는 구현 시 상수로)                                                                               | 🟢     |
| F9-11 | month 파라미터 (F3)     | `YYYYMM` 형식·실존 월 검증                                                                                                                                   | 🟢     |
