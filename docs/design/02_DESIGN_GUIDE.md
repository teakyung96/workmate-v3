# 03. 디자인 가이드 (Design Guide)

> ⚠️ **v3 안내**: 이 문서는 v2에서 계승한 **컬러·타이포·컴포넌트 톤의 기준값**이다.
> v3의 **스타일링 방식**은 [00_DESIGN_SYSTEM](00_DESIGN_SYSTEM.md)(shadcn-vue + Tailwind v4)을 따른다 —
> 아래 색·타이포 값은 그대로 유지하되, **shadcn-vue 테마 토큰(CSS 변수)으로 매핑**해 사용한다.
> (문서 내부 링크 일부는 v2 경로 기준이라 새 구조와 다를 수 있음)

- **작성일**: 2026-07-14 (v2), v3 편입 2026-07-21
- **상태**: v3 테마 기준값으로 채택
- **상위 문서**: [workmate-design.md](workmate-design.md)
- **관련 문서**: [01 기능 명세서](01_FEATURE_SPEC.md) · [02 화면 설계서](02_SCREEN_DESIGN.md)

모든 화면이 따르는 시각 표준. 구조(UX)는 Claude.ai, 시각 톤은 shadcn/ui를 참고하되 픽셀 복제는 하지 않는다.
**테마는 라이트 한 벌만 지원**한다 — 단, 모든 색은 CSS 변수(토큰)로만 사용해 다크 모드를 나중에 추가할 수 있게 한다.

## 0. 운영 원칙

- 토큰(CSS 변수)은 `static/css/common.css`의 `:root`에서만 정의한다. 페이지 CSS는 **토큰을 참조만** 하고 새 색상값을 만들지 않는다.
- 인라인 `<style>` 블록 금지, 클래스명은 kebab-case (전역 규칙).
- 기존 `common.css`의 5개 토큰(`--bg-primary` 등)은 본 가이드 체계로 **확장·개명**한다 (§7 마이그레이션).

## 1. 컬러 토큰

### 1.1 포인트 컬러 — 딥 블루 (확정)

| 토큰                  | 값        | 용도                                               |
| --------------------- | --------- | -------------------------------------------------- |
| `--primary`           | `#2563EB` | 주 버튼, 활성 메뉴, 사용자 말풍선, 링크, 포커스 링 |
| `--primary-hover`     | `#1D4ED8` | 주 버튼/링크 호버                                  |
| `--primary-soft`      | `#EFF6FF` | 활성 항목 배경, 선택 강조 배경 (blue-50)           |
| `--primary-soft-text` | `#1E40AF` | soft 배경 위 텍스트 (blue-800)                     |

### 1.2 뉴트럴 (쿨 그레이 — 딥 블루와 조화)

| 토큰           | 값        | 용도                                |
| -------------- | --------- | ----------------------------------- |
| `--bg`         | `#F3F4F6` | 페이지 배경 (gray-100)              |
| `--bg-sidebar` | `#F9FAFB` | 사이드바 배경 (gray-50)             |
| `--surface`    | `#FFFFFF` | 카드·모달·입력창 배경               |
| `--border`     | `#E5E7EB` | 구분선·테두리 (gray-200)            |
| `--text`       | `#111827` | 본문 텍스트 (gray-900)              |
| `--text-muted` | `#6B7280` | 보조 텍스트·플레이스홀더 (gray-500) |
| `--text-faint` | `#9CA3AF` | 비활성·타임스탬프 (gray-400)        |

### 1.3 시맨틱 컬러 (상태 표시 — 포인트 컬러와 별개)

| 토큰                           | 값                    | 용도                                          |
| ------------------------------ | --------------------- | --------------------------------------------- |
| `--success` / `--success-soft` | `#16A34A` / `#F0FDF4` | 성공 토스트, 확정 뱃지(✅), 체크섬 통과       |
| `--warning` / `--warning-soft` | `#D97706` / `#FFFBEB` | "확인 필요" 뱃지, 체크섬 실패 경고, 폴백 안내 |
| `--danger` / `--danger-soft`   | `#DC2626` / `#FEF2F2` | 삭제 버튼, 에러 메시지, 계정 잠금 안내        |

> 규칙: 성공/경고/위험 표현에 `--primary`를 쓰지 않는다. 반대로 강조·선택 표현에 시맨틱 컬러를 쓰지 않는다.

## 2. 타이포그래피

| 항목           | 값                                                                                   |
| -------------- | ------------------------------------------------------------------------------------ |
| 폰트 패밀리    | `"Outfit", "Noto Sans KR", sans-serif` (기존 유지 — 영문 Outfit + 한글 Noto Sans KR) |
| 기본 크기/행간 | `16px` / `1.6` (채팅 본문 기준)                                                      |

**타입 스케일** (이외 크기 임의 사용 금지):

| 토큰          | 크기/굵기  | 용도                       |
| ------------- | ---------- | -------------------------- |
| `--font-h1`   | 24px / 700 | 페이지 제목 (예: "영수증") |
| `--font-h2`   | 18px / 600 | 섹션 제목, 방 제목         |
| `--font-body` | 16px / 400 | 본문, 채팅 메시지          |
| `--font-sm`   | 14px / 400 | 목록 보조 정보, 폼 라벨    |
| `--font-xs`   | 12px / 500 | 뱃지, 타임스탬프, 캡션     |

- 숫자 정렬이 필요한 곳(영수증 금액·이력 테이블)은 `font-variant-numeric: tabular-nums`.
- AI 응답 마크다운: 코드블록은 `--bg-sidebar` 배경 + `--border` 테두리, 등폭 폰트 `Consolas, monospace`.

## 3. 간격·라운드·그림자

**간격 스케일** — 4px 배수만 사용: `4 / 8 / 12 / 16 / 24 / 32 / 48`

| 용도                           | 값                        |
| ------------------------------ | ------------------------- |
| 컴포넌트 내부 패딩 (버튼·입력) | 세로 8~12 / 가로 16       |
| 카드 내부 패딩                 | 24                        |
| 섹션 간 간격                   | 32                        |
| 사이드바 폭                    | 260px (접힘 기준은 02 §0) |

**라운드**:

| 토큰          | 값   | 용도                  |
| ------------- | ---- | --------------------- |
| `--radius-sm` | 6px  | 뱃지, 인풋 내부 요소  |
| `--radius`    | 10px | 버튼, 입력창, 목록 행 |
| `--radius-lg` | 14px | 카드, 모달, 말풍선    |

**그림자** — 절제 (2단계만):

| 토큰          | 값                             | 용도           |
| ------------- | ------------------------------ | -------------- |
| `--shadow`    | `0 1px 3px rgb(0 0 0 / 0.08)`  | 카드           |
| `--shadow-lg` | `0 8px 24px rgb(0 0 0 / 0.12)` | 모달, 드롭다운 |

## 4. 공통 컴포넌트 톤

### 4.1 버튼

| 종류      | 스타일                                             | 예                                |
| --------- | -------------------------------------------------- | --------------------------------- |
| Primary   | `--primary` 배경 + 흰 글자, 호버 `--primary-hover` | [분석하기], [로그인], [+ 새 채팅] |
| Secondary | `--surface` 배경 + `--border` 테두리 + `--text`    | [다시 선택], [취소]               |
| Danger    | `--danger` 배경 + 흰 글자 — **파괴적 행동 전용**   | [삭제], [초기화]                  |
| Ghost     | 배경 없음 + `--text-muted`, 호버 시 `--bg`         | 아이콘 버튼(📋, 🗑)                |

- 비활성: `opacity: 0.5` + `cursor: not-allowed`.
- 처리 중: 버튼 내 `<loading-spinner>` + 라벨 유지 ("분석 중…").

### 4.2 입력 (input·textarea·select)

- 기본: `--surface` 배경, `--border` 1px, `--radius`.
- 포커스: `--primary` 테두리 + `box-shadow: 0 0 0 3px var(--primary-soft)` (포커스 링 — 키보드 접근성 필수).
- 에러: `--danger` 테두리 + 아래 `--font-xs` 에러 문구.

### 4.3 뱃지 (02 화면 설계의 상태 표시)

| 뱃지                    | 스타일                                                               |
| ----------------------- | -------------------------------------------------------------------- |
| ✅ 확정 / 검증 통과     | `--success-soft` 배경 + `--success` 글자                             |
| ⚠ 확인 필요 / 검증 실패 | `--warning-soft` 배경 + `--warning` 글자                             |
| AUTO 자동 선택          | `--primary-soft` 배경 + `--primary-soft-text` 글자                   |
| 공개/비공개 (가이드)    | 공개: `--primary-soft` / 비공개: `--bg` + `--text-muted`             |
| 📎 출처 (RAG)           | `--primary-soft` 배경 + `--primary-soft-text`, 클릭 가능 (링크 역할) |

### 4.4 채팅 말풍선

| 요소          | 스타일                                                                         |
| ------------- | ------------------------------------------------------------------------------ |
| 사용자 (우측) | `--primary` 배경 + 흰 글자, `--radius-lg`, 우하단 모서리만 4px                 |
| AI (좌측)     | `--surface` 배경 + `--border`, `--radius-lg`, 좌하단 모서리만 4px, 최대 폭 85% |
| 스트리밍 커서 | `--primary` 세로 막대, `1s` 깜빡임. `prefers-reduced-motion` 시 정지           |

### 4.5 모달·토스트

- 모달: 중앙, `--radius-lg` + `--shadow-lg`, 배경 딤 `rgb(0 0 0 / 0.4)`. 파괴적 확인 모달의 확정 버튼은 Danger.
- 토스트: 우측 상단, 3초 자동 소멸 (02 §9). 성공 `--success` / 실패 `--danger` 좌측 스트라이프.

## 5. 접근성 최소 기준

- 텍스트 대비 4.5:1 이상 (본 팔레트는 충족 — `--text-muted`까지 흰 배경 대비 4.6:1).
- 모든 인터랙티브 요소에 키보드 포커스 링 표시 (§4.2 포커스 규칙 공통 적용).
- 아이콘 단독 버튼(📋·🗑)은 `aria-label` 필수.
- 애니메이션은 `prefers-reduced-motion: reduce` 존중.

## 6. 다크 모드 대비 규칙 (지금은 미지원)

- 색상값을 컴포넌트 CSS에 직접 쓰지 않고 **반드시 토큰 경유** — 다크 추가 시 `:root` 재정의만으로 전환 가능하게.
- `#FFF`·`#000` 하드코딩 금지 (`--surface`·`--text` 사용).
