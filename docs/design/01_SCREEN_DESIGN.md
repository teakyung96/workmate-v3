# 02. 화면 설계서 (Screen Design)

- **작성일**: 2026-07-14
- **상태**: 사용자 검토 대기
- **상위 문서**: [workmate-design.md](workmate-design.md)
- **관련 문서**: [01 기능 명세서](01_FEATURE_SPEC.md) · [03 디자인 가이드](03_DESIGN_GUIDE.md) · [04 API·DB 설계서](04_API_DB_SPEC.md) · [05 공통 요구사항](05_COMMON_REQUIREMENTS.md)

이 문서는 7개 화면의 레이아웃·구성요소·이동 흐름을 정의한다.
시각 토큰(색·폰트·간격)은 03 디자인 가이드를 따르고, 여기서는 **구조와 동작**만 다룬다.
각 구성요소 옆 `(F#-##)`는 01 기능 명세서의 요구사항 ID다.

---

## 0. 공통 레이아웃 — 사이드바 통합형

로그인·회원가입을 제외한 모든 화면은 아래 공통 틀 안에서 **중앙 본문만 교체**된다.

```
┌─────────────┬──────────────────────────────────────┐
│  Workmate   │                                      │
│             │                                      │
│ [+ 새 채팅]  │                                      │
│ ─────────── │                                      │
│ 채팅 목록     │         중앙 본문 영역                 │
│  · 월간 보고… │   (채팅 / 영수증 / 가이드 / 관리자)      │
│  · 회의록 정… │                                      │
│  · …        │                                      │
│ ─────────── │                                      │
│ 🧾 영수증    │                                      │
│ 📚 가이드    │                                      │
│ ⚙ 관리자     │  ← ROLE_ADMIN만 노출 (F6-04)          │
│ ─────────── │                                      │
│ 👤 김태경 ▾  │  ← 클릭: 로그아웃 메뉴                  │
└─────────────┴──────────────────────────────────────┘
```

### 사이드바 구성요소

| 요소                       | 동작                                                                                                   | 근거         |
| -------------------------- | ------------------------------------------------------------------------------------------------------ | ------------ |
| 로고 `Workmate`            | 클릭 시 채팅 홈(빈 채팅)으로 이동                                                                      |              |
| **[+ 새 채팅]** 버튼       | 빈 채팅 화면으로 전환. 이 시점에 방 생성 안 함                                                         | F2-01, F2-02 |
| 채팅 목록                  | 본인 방 최신순. 현재 방은 활성(포인트 컬러) 표시. 항목 호버 시 삭제(🗑) 아이콘 노출 → 확인 후 논리 삭제 | F2-03, F2-04 |
| 🧾 영수증 / 📚 가이드 메뉴 | 중앙 본문을 해당 기능으로 전환. 현재 위치 활성 표시                                                    |              |
| ⚙ 관리자 메뉴              | ROLE_ADMIN에게만 렌더링 (서버 측 접근 차단 별도)                                                       | F6-04        |
| 👤 사용자 영역             | 이름 표시, 클릭 시 로그아웃                                                                            | F1-09        |

### 반응형 규칙

- 기준 폭 1024px 미만(태블릿): 사이드바는 햄버거(☰) 버튼으로 접힘 — 오버레이로 열림.
- 모바일(640px 미만): 동일 + 본문 여백 축소. Flexbox/Grid 기반 (전역 규칙).

### 구현 메모

- 사이드바는 Thymeleaf Fragment(`fragments/sidebar.html`)로 분리 — 2개 이상 화면 공유 (02 백엔드 가이드 §8). 인터랙션 JS는 `static/js/sidebar.js` (03 가이드 §5.6).
- 페이지 스크립팅은 Vue 3 CDN 패턴, 마운트 id는 `{페이지명}App` (03 가이드 §5.3).

## 3. 채팅 (메인, `/`) — 기본 화면

로그인 직후·[+ 새 채팅] 클릭 시의 **빈 채팅 상태**와 **대화 중 상태** 두 가지.

### 3-A. 빈 채팅 상태 (방 미생성)

```
┌─────────────┬──────────────────────────────────────┐
│ (사이드바)    │                                      │
│             │        무엇을 도와드릴까요?              │
│             │                                      │
│             │   ┌────────────────────────────────┐ │
│             │   │ 메시지를 입력하세요…               │ │
│             │   │              [모델 ▾] [RAG ○] ➤ │ │
│             │   └────────────────────────────────┘ │
└─────────────┴──────────────────────────────────────┘
```

### 3-B. 대화 중 상태

```
┌─────────────┬──────────────────────────────────────┐
│ (사이드바)    │  월간 보고서 요약                       │ ← 방 제목
│  · 월간 보…● │ ├──────────────────────────────────── │
│             │                    ┌───────────────┐ │
│             │                    │ 지난달 영수증   │ │ ← 사용자 말풍선(우측)
│             │                    │ 총액 알려줘     │ │
│             │                    └───────────────┘ │
│             │  ┌─────────────────────────────┐     │
│             │  │ 6월 영수증은 총 12건,         │     │ ← AI 응답(좌측, 마크다운)
│             │  │ 합계 486,200원입니다.▌       │     │ ← 스트리밍 커서
│             │  │ 📎 출처: 경비처리 가이드       │     │ ← RAG 출처 뱃지 (F4-07)
│             │  └─────────────────────────────┘     │
│             │   ┌────────────────────────────────┐ │
│             │   │ 메시지를 입력하세요…              │ │
│             │   │              [모델 ▾] [RAG ●] ➤ │ │
│             │   └────────────────────────────────┘ │
└─────────────┴──────────────────────────────────────┘
```

| 요소                | 동작                                                                                                                    | 근거         |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------- | ------------ |
| 입력창              | 멀티라인(Shift+Enter 줄바꿈, Enter 전송). 빈 입력 시 전송(➤) 비활성                                                     | F2.3         |
| 전송                | 첫 전송이면 방 생성 + 사이드바 목록 갱신. 스트리밍 중엔 입력 비활성                                                     | F2-02        |
| AI 응답             | SSE 글자 단위 표시 + 깜빡이는 커서, 완료 후 마크다운 렌더링                                                             | F2-06, F2-07 |
| [모델 ▾] 드롭다운   | 공통코드 `AI_MODEL` 조회로 구성. 기본값은 설정 모델                                                                     | F7-04, F5-05 |
| [RAG ○/●] 토글      | 켜면 질문마다 유사 청크 검색 포함. 상태는 방 단위 유지                                                                  | F4-05        |
| 출처 뱃지           | RAG 답변에 항상 표시. 클릭 → 해당 가이드 상세로 이동                                                                    | F4-07        |
| 응답 중단 시        | 수신분 보존 + "응답이 중단되었습니다" + [재시도] 버튼                                                                   | F2.3         |
| 관련 청크 없음(RAG) | "문서에서 관련 내용을 찾지 못했습니다" 안내 + **[일반 모드로 다시 질문]** 버튼 (01 F4.3의 열린 결정 → 이 방식으로 확정) | F4.3         |

### 상태별 화면

## 4. 영수증 (`/receipt`) — [분석]/[이력] 탭

한 화면 + 탭 전환 (F3-13). 기본 탭은 [분석].

### 4-A. [분석] 탭 — 업로드 전

```
┌─────────────┬──────────────────────────────────────┐
│ (사이드바)    │  영수증   [ 분석 ] [ 이력 ]            │
│             │ ├──────────────────────────────────── │
│             │   ┌────────────────────────────────┐ │
│             │   │      🧾                        │ │
│             │   │  영수증 이미지를 끌어다 놓거나      │ │
│             │   │  클릭해서 선택하세요               │ │
│             │   │  (jpg / png / webp, 10MB 이하)  │ │
│             │   └────────────────────────────────┘ │
└─────────────┴──────────────────────────────────────┘
```

### 4-B. [분석] 탭 — 이미지 선택 후 (분석 전)

```
│             │   ┌───────────────┐                  │
│             │   │               │   [ 분석하기 ]     │
│             │   │  이미지        │   [ 다시 선택 ]    │
│             │   │  미리보기       │                  │
│             │   │               │                  │
│             │   └───────────────┘                  │
```

### 4-C. [분석] 탭 — 분석 결과 (원본과 나란히)

```
│             │   ┌───────────────┐ ┌──────────────────────┐
│             │   │               │ │ ✅ 롯데법인카드 자동 선택 │ ← AUTO 뱃지 (F3-04)
│             │   │  원본 이미지    │ │                      │
│             │   │  (클릭 시 확대) │ │ 금액     [ 48,600 ]📋 │
│             │   │               │ │ 사업자번호 [1234567890]📋│
│             │   │               │ │  ⚠ 검증 실패 — 확인 필요 │ ← 체크섬 경고 (F3-07)
│             │   │               │ │ 결제일    [ 20260701 ]📋 │
│             │   └───────────────┘ │                      │
│             │                     │ 수정하면 자동 반영됩니다   │ ← UPDATE 안내 (F3-08a)
│             │                     └──────────────────────┘
```

**MANUAL 케이스** (롯데카드 0건/복수 건, F3-05·F3-08b): 우측 폼 대신 결제 건 선택 리스트가 먼저 표시.

```
│             │                     │ ⚠ 결제 건을 선택해주세요   │
│             │                     │ ○ 신한카드   32,000원 7/1 │
│             │                     │ ○ 롯데법인카드 8,600원 7/1 │
│             │                     │ ○ 롯데법인카드 8,000원 7/1 │
│             │                     │      [ 이 결제 건 선택 ]  │
```

| 요소            | 동작                                                                     | 근거           |
| --------------- | ------------------------------------------------------------------------ | -------------- |
| 업로드 영역     | 드래그앤드롭 + 클릭 선택. 형식·크기 클라이언트 선검증                    | F3-01          |
| 이미지 미리보기 | 선택 즉시 표시 — 파일명 텍스트만 표시 금지                               | F3-01a         |
| [분석하기]      | 분석 + **이력 자동 저장**. 완료 후 결과 화면 전환. 처리 중 스피너+비활성 | F3-08          |
| 결과 폼         | 원본 이미지와 나란히. 값 수정 시 저장된 건 UPDATE                        | F3-01b, F3-08a |
| 📋 복사 버튼    | 필드별 클립보드 복사 + "복사됨" 토스트                                   | F3-11          |
| 체크섬 경고     | `biz_no_valid=false`면 필드 아래 경고 (저장은 유지)                      | F3-07          |
| 결제 건 선택    | 선택 순간 UPDATE로 확정                                                  | F3-08b         |

### 예외 상태 화면 ([분석] 탭)

| 상황              | 표시                                                                                   | 근거         |

| 영수증 아님       | 모달: "영수증으로 인식되지 않았습니다" + [다시 업로드] [그래도 수동 입력]. 이력 미저장 | F3-02a, F3.3 |
